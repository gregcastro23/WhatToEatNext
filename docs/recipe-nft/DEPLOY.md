# Recipe NFT ‚Äî deploy & wiring checklist

The off-chain mint mechanic (cost quote, ESMS debit, ledger) ships gated. Minting goes
live on-chain once these steps complete. Off-chain ledger remains authoritative.

## 1. Deploy the protocol (Base Sepolia)

In the contracts repo (`~/Desktop/EthGlobalHackathon/AlchmAgentsETH-main/contracts`):

```bash
# Compute the hashes (license manifest is in this repo):
bun run scripts/recipe-nft/compute-deploy-hashes.ts \
  --work <canonical-work-file> --evidence <copyright-cert-file>

# Set env, then deploy + register the VA 2-434-962 anchor:
export DEPLOYER_PRIVATE_KEY=...        # funded Base Sepolia deployer
export RIGHTS_HOLDER=0x553C2a3f193d5E7F41cF50cEB32069dbc6951931
export ALCHM_WORK_HASH=0x...
export ALCHM_EVIDENCE_HASH=0x...
export ALCHM_LICENSE_HASH=0x...        # keccak256 of ALCHM-RECIPE-LICENSE-v1.md
export ALCHM_LICENSE_URI=...           # public URI hosting that manifest

forge script script/DeployRecipeProtocol.s.sol:DeployRecipeProtocol \
  --rpc-url base_sepolia --broadcast --verify
```

The run prints `AlchmRightsRegistry`, `RecipeRegistry`, and the genesis `rightsId`.

## 2. Authorize the sponsor (backend) wallet

Minting is backend-sponsored: a server wallet mints on the user's behalf. Authorize it as
an operator under the rights anchor (rights-holder action):

```
AlchmRightsRegistry.setOperator(rightsId, <RECIPE_MINTER_address>, true)
```

## 3. Configure WTEN env

```
NEXT_PUBLIC_RECIPE_NFT_CHAIN=base-sepolia
NEXT_PUBLIC_RECIPE_REGISTRY_ADDRESS=0x...   # from step 1
NEXT_PUBLIC_RIGHTS_REGISTRY_ADDRESS=0x...   # from step 1
NEXT_PUBLIC_ALCHM_RIGHTS_ID=0x...           # genesis rightsId from step 1
NEXT_PUBLIC_ALCHM_LICENSE_URI=...           # same URI as ALCHM_LICENSE_URI
NEXT_PUBLIC_RECIPE_NFT_ENABLED=true         # flips recipeNftEnabled() on
RECIPE_MINTER_PRIVATE_KEY=0x...             # sponsor wallet (or reuse MINTER_PRIVATE_KEY)
BASE_SEPOLIA_RPC_URL=...                     # optional; avoids public-RPC rate limits
```

## 4. Apply the DB migration

`database/init/55-recipe-nft-mint-schema.sql` ‚Äî seeds the `recipe-nft-mint` shop item and
creates `recipe_nft_mints`. Apply it to the WTEN database.

## 5. Display metadata + hero image (DONE ‚Äî served from our own infra, no IPFS)

The mint route now generates the hero image via the live nanobanana pipeline and sets
absolute `contentURI`/`metadataURI` (no more empty-URI revert):

- `GET /api/recipes/nft/metadata/{contentHash}` ‚Äî ERC-721 JSON (the `tokenURI` target).
- `GET /api/recipes/nft/content/{contentHash}` ‚Äî canonical content envelope.
- `GET /api/recipes/featured/hero-image` ‚Äî hero image for the Promo card.

These build absolute URLs from `getSelfBaseUrl()`, so **set `NEXT_PUBLIC_SITE_URL=https://alchm.kitchen`**
in prod or the URIs will be `localhost`. The hero image is generated by **Cloudflare
Workers AI (SDXL)** and uploaded to **R2** (`assets.alchm.kitchen/recipes/nft/{id}.png`) ‚Äî
durable by design. Needs `CLOUDFLARE_ACCOUNT_ID`, `CLOUDFLARE_API_TOKEN`, and the `R2_*`
credentials (same as `scripts/generate-recipe-images.ts`).

## 6. Pending items before flipping on
